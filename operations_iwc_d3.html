Observability patterns

Monitor from the perspective of the user

Have artificial users as a control

Trace interactions through the system

Be able to flexibly divide - by stack, by stage, by site, by type

Use precise language

Monitor the whole and the parts - from one place

Place observabiliity on the architecure agenda

Write and prioritise operator stories

If you must send alerts - send context too

Maps, traffic lights, gauges - time is a vital dimension

Be ready for deep inspection

Own the security policy

Architect things that are easy to monitor

Make events self-describing

Treat an absence of oddities as a shortfall in monitoring
